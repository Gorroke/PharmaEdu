## 8. 보훈 50%면제 + 암환자 비즈팜 버그 회피

### 8-1. 비즈팜 BUG-002 설명 (CH08 원본 §9-2)

```
위치: Ppre1000__.frm 라인 9732
코드: Insur_Self_Amt = Insur_Self_Amt = Int((...))
문제: VB6에서 A = B = C는 B와 C를 비교한 Boolean(-1/0)을 A에 대입
영향: 암환자(chkCa=1) + M50(50%감면) + 조합/보험 가입자 → 본인부담금 0원 또는 -1원
```

### 8-2. TS 구현 회피 여부: ✓ 회피됨

`veteran.ts:calcVeteran():L326-339`:
```typescript
// 일반 보훈 감면 (M30/M50/M60/M81/M83/M90)
else {
  const basisAmt = totalPrice - mpvaPrice;
  if (bohunRate === 30 || bohunRate === 50 || bohunRate === 60 || bohunRate === 90) {
    userPrice = trunc10(basisAmt * insuRate / 100);
  } else {
    userPrice = trunc100(basisAmt * insuRate / 100);
  }
}
```

M50(bohunRate=50)에 대해 `trunc10(basisAmt * insuRate / 100)`으로 정상 산식이 적용된다. VB6 이중등호 패턴이 재현되지 않는다.

### 8-3. 추가 확인 필요 사항

BUG-002의 발동 조건은 **암환자(chkCa=1) + M50 + 조합/보험 가입자**의 조합이다. 현재 `veteran.ts`는 암환자 플래그(`chkCa`)와 M50의 **조합 케이스** 분기를 별도로 갖고 있지 않다.

CH08 원본에 따르면 암환자 여부에 따라 별도 본인부담 산식이 존재하는데, 이 분기가 현재 `veteran.ts`에 없다.

- [🟠 Insufficient / High] **암환자 + M50 조합 분기 없음**: BUG-002 회피는 됐으나, 암환자(`chkCa=1`) 상태에서 M50 보훈 감면 시 적용해야 할 별도 산식이 `veteran.ts`에 없다. `calcVeteran()`에서 암환자 플래그를 받지도 않는다 (`CalcOptions`에 `chkCa` 또는 `isCancer` 필드 없음). CH08 §9-2에서 "50%감면 본인부담금을 정상 산식으로 구현한다"는 지침은 충족했으나, 암환자 상태에서의 별도 로직 존재 여부를 C# 원본에서 추가 확인 필요.

## 12. 기타 관찰 사항

1. `seasonal.ts`의 로직 자체(단가, 코드 매핑, 비대면 제외)는 정확하나 날짜 범위 정의가 CH08 원본보다 좁다.
2. `drug-648.ts`의 보훈 면제 목록은 `['M10', 'M83', 'M82']`(L39)인데, CH08 §4-3에서 M61/M20도 가산 **적용** 대상으로 명시되어 있어 면제 목록이 올바르게 제한됨을 재확인했다.
3. `HolidayTable.cs`에 2026년 공휴일이 포함되어 있으나 `seasonal.ts`는 2026 명절을 지원하지 않는다. 2026 설날(02-17), 추석(09-24) 코드가 필요하다면 HOLIDAY_TABLE 확장 필요.
4. `safety-net.ts`의 본인부담상한제 로직은 사후정산 방식을 기본으로 하여 `cumulativeUserPrice` 미제공 시 미적용 처리하는데(L185-191), 이는 약국 청구 현실에 부합한다.
5. DateThresholds에서 향후 필요성이 낮은 레거시 항목(2007년, 2015년 등)은 현재 TS 엔진에 구현되지 않았으나, PharmaEdu가 현행 기준 교육용이라면 수용 가능하다.
