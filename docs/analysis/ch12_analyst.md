## 1. 챕터 개요

- **챕터 제목**: 보훈 약국 약제비 청구
- **핵심 주제**: 보훈병원 발행 처방전에 따라 조제하는 원외약국이 건강보험 심사평가원(심사위탁) 또는 보훈위탁진료 작성요령에 따라 약제비를 청구하는 규칙 전반을 정리한다. 환자 구분(국비/면제/감면/일반), 감면율 체계, `보훈청구액 = standard_copay − veteran_copay` 계산식, 공상등구분 코드 매핑, MT038 사용 조건을 핵심 규격으로 삼는다.
- **다루는 계산 로직 범위**:
  - 보훈 3자배분: `총액 = insuPrice + userPrice(veteran_copay) + mpvaPrice(보훈청구액)` (CH12 §5.5, §4.3)
  - 감면율 적용 모수: 건강보험/의료급여기금 부담분을 제외한 환자 본인부담 영역 (CH12 §4.3, P16 제11조제7항)
  - 두 청구 체계 분리: 보훈병원 업무처리(공상등구분 3/5/6/J) vs 보훈위탁(0(E)/6+A/4/7) (CH12 §5.3, §5.4)
  - MT038 사용 조건: 약국은 보훈위탁 특정 상황에서만 'A' 또는 '2' 사용, 보훈병원 업무처리 기준 약국은 미사용 (CH12 §5.2, §5.6)
  - 보훈국비환자(4/7): MT038='2' 적용 (2013-01-01~), MT038='1' 삭제 (2018-01-01~) (CH12 §5.6)
  - 자가처치 치료재료 26품목 BZ401001~BZ499002 월/일 상한 (CH12 §8)
  - 위탁수수료 약국 66원/건, 이의신청 90일 이내 (CH12 §10.3, §10.3)
  - 관련 법령: P16 운영규정(2025.07.02), P15 업무처리 기준(2024.10.22), P14 보훈병원 의료수가 기준(2026.01.01), P08 심사평가원 심사위탁 별표2, P10 심사·평가업무 위탁 계약서

### 미구현 영역

- `두 청구 체계 분리 모듈` — CH12 §5.3, §5.4 참조. 보훈병원 업무처리 기준(공상등구분 3/5/6/J)과 보훈위탁진료 작성요령 기준(0(E)/6+A/4/7)을 별도 분기하는 함수·상수가 코드베이스에서 확인되지 않음.
- `MT038 사용 조건 처리 모듈` — CH12 §5.2, §5.6 참조. MT038='A'(도서벽지 60% 감면 위탁 약국), MT038='2'(국비환자 4/7), MT038='1' 삭제(2018~) 분기를 담당하는 전용 함수 없음.
- `자가처치 치료재료 상한관리 모듈` — CH12 §8 참조. BZ401001~BZ499002 26품목의 월/일/주 상한 검사 및 본인부담 전환 로직 없음.
- `위탁수수료 66원 집계` — CH12 §10.4 참조. 약국 청구건당 66원 계산 및 집계 모듈 없음.

## 4. 누락 항목 (Missing)

- [🔴 Missing / Critical] **두 청구 체계 분리**: 보훈병원 업무처리 기준(공상등구분 3/5/6/J)과 보훈위탁진료 작성요령 기준(공상등구분 0(E)/6/4/7 + MT038)을 분리 처리하는 분기 모듈이 전혀 없다. 두 체계를 혼용하면 공상등구분 코드 오기재로 청구 반송 발생. (CH12 §5.3, §5.4)

- [🔴 Missing / Critical] **보훈위탁진료 약국 공상등구분 4/7 + MT038='2' 처리**: 일부본인부담대상 전상군경등의 국비질환 외 타질환 조제(2013.01.01~)에 대해 공상등구분 7→4, MT038='2' 기재 규칙을 구현하는 함수가 없다. (CH12 §5.6)

- [🔴 Missing / Critical] **MT038='1' 날짜 기준 삭제 분기**: 2018.01.01 진료분부터 MT038='1'을 사용하지 않도록 날짜 기반 분기를 해야 하나, 해당 로직이 전혀 없다. (CH12 §5.6)

- [🔴 Missing / Critical] **도서벽지 60% 감면 MT038='A' 처리**: 보훈위탁진료 약국에서 도서벽지 내 60% 감면 환자에게 MT038='A'를 기재하는 로직이 없다. (CH12 §5.4)

- [🟠 Missing / High] **급여/비급여/특수재료 분리 처리**: P08 1.나의 위탁 제외 범위(비급여 진료비, 한방 진료비, 국고지원금, 특수재료 및 관련 행위료)를 분리하는 플래그·필드가 없다. (CH12 §3.4)

- [🟠 Missing / High] **자가처치 치료재료 26품목 상한관리**: BZ401001~BZ499002 26품목의 월/일/주 단위 상한, 적응증, 혼용 금지 검사 모듈이 없다. (CH12 §8)

- [🟠 Missing / High] **국가부담 제한 항목 제외 처리**: P14 제9조·P16 제11조제7항의 제외 항목(비만 관련, 증명서 수수료, 치료재료대 중 요양급여 외, 보철구대, 상급병실, 자가처치 상한 초과분)에 대한 감면 제외 플래그·필터가 없다. (CH12 §6.3)

- [🟠 Missing / High] **P05 예방진료 감면비율 별도 분기**: P05는 대상자별로 진료비 감면비율과 예방진료 감면비율을 구분한다(예: 참전유공자 90%/60%). 현재 구현은 단일 감면율만 반환하며 예방진료 여부 분기가 없다. (CH12 §4.2)

- [🟡 Missing / Medium] **명세서 비고 문구 생성**: P09 별도산정 약제·치료재료(자가처치용, IVUS SONO용, 방광경검사용 등)에 대한 명세서 비고 문구 생성 로직이 없다. (CH12 §7.1)

- [🟡 Missing / Medium] **위탁수수료 집계(약국 66원/건)**: 대량 정산·회계 모듈과의 연결을 위한 수수료 집계 처리가 없다. (CH12 §10.4)

- [🟡 Missing / Medium] **이의신청 기한 관리**: 처분일로부터 90일 이내 이의신청, 심평원 60일 이내 결정 등 기한 관리 기능이 없다(운영 기능이므로 계산 엔진과는 분리 가능). (CH12 §10.3)

## 5. 부족 항목 (Insufficient)

- [🟠 Insufficient / High] **공상등구분 3/5/6/J ↔ BohunCode M코드 명시적 매핑 없음**: `veteran.ts`의 `getBohunRate()`는 M30/M50/M60/M83 등 M코드를 받아 처리하지만, 실제 EDI 명세서의 공상등구분 '3'/'5'/'6'/'J'를 M30/M50/M60/M83으로 변환하는 입력 매핑 함수가 없다. 호출 측(상위 레이어)에서 처리할 것으로 보이나 코드베이스에서 확인되지 않는다. (CH12 §5.3, `src/lib/calc-engine/modules/insurance/veteran.ts`)

- [🟠 Insufficient / High] **P06 13번(공단 임직원 50%)/14번(병원장 재량 50% 이하) 분기 미처리**: `getBohunRate()`는 13번·14번에 해당하는 별도 코드나 분기가 없다. 이 두 항목은 원외약국 1차 구현 범위에서 제외 가능하나, 범위 제외를 명시하는 주석·가드 조건이 없다. (CH12 §4.1, `src/lib/calc-engine/modules/insurance/veteran.ts:getBohunRate():L128-146`)

- [🟠 Insufficient / High] **위탁(isMPVBill)/비위탁 절사 방식의 근거 문서 불일치**: `calcMpvaPrice()`는 위탁 시 `trunc10(총액 × 감면율%)`, 비위탁 시 역산 방식을 쓴다. CH12 §4.3은 `standard_copay − veteran_copay` 공식을 명시하나, trunc10 절사단위 자체를 "미확정"으로 표시한다. 10원 절사가 향후 확정 규칙과 다를 경우 계산 오류 발생. (CH12 §4.3, `src/lib/calc-engine/modules/insurance/veteran.ts:calcMpvaPrice():L210-226`)

- [🟡 Insufficient / Medium] **M61 이중감면 역산 로직 검증 부족**: `calcVeteran()`의 M61 처리(L314-324)는 `normalUser - userPrice`로 mpvaPrice를 재산출한다. CH12 원문은 M61(고엽제)을 "역산 방식"으로만 서술하며 수식을 명시하지 않는다. C# 원본 CopaymentCalculator.cs와의 수식 대조가 필요하다. (`src/lib/calc-engine/modules/insurance/veteran.ts:calcVeteran():L314-324`)

- [🟡 Insufficient / Medium] **보훈병원 업무처리 vs 보훈위탁 진입 분기 없음**: `copayment.ts`의 보훈 분기(L85-93)는 `bohunCode.startsWith('M')` 또는 `insuCategory === 'G'`이면 `calcVeteran()`을 호출한다. 그러나 해당 호출이 두 청구 체계 중 어느 경로인지 구분하지 않는다. `calcVeteran()` 내부도 두 체계를 구분하지 않으므로, 명세서 필드 생성 시 오기재 위험이 있다. (`src/lib/calc-engine/copayment.ts:calcCopayment():L85-93`)

- [🟡 Insufficient / Medium] **세부 보훈코드 M10/M20/M61/M81/M82/M83 직접 근거 부재**: `veteran.ts` 상단 주석에 M10~M90 코드 목록이 있으나, 이 코드들의 원천 근거가 CH12 PDF에는 없고 "추가 검증 후 구현 범위"로 분류된다. 코드 매핑의 법령 근거를 주석에 명시할 필요가 있다. (CH12 §11.2, `src/lib/calc-engine/modules/insurance/veteran.ts:L8-26`)

## 6. 기타 관찰 사항

- **`별표 4` 비급여 품목 마스터 미완성**: CH12 §7.3은 P12 PDF 본문이 `"엑셀자료 별도붙임"` 한 줄뿐이라고 명시한다. 따라서 비급여 약제·치료재료 자동판정 테이블을 현재 PDF 세트만으로는 구성할 수 없으며, 별도 엑셀 원본 확보가 선행되어야 한다. 계산 엔진에서 비급여 판정을 지원하려면 이 데이터 확보가 필수이다.

- **보훈청구액 절사단위 미확정 플래그 필요**: CH12 §4.3은 보훈청구액의 10원 절사/올림/사사오입이 현재 PDF만으로 확정되지 않는다고 명시한다. `calcMpvaPrice()`는 `trunc10()`을 하드코딩으로 사용 중이다. 향후 정책 확정 시 단 한 곳만 수정할 수 있도록 `BOHUN_TRUNC_UNIT` 같은 정책 상수를 별도로 선언하는 것을 권장한다.

- **3자배분 항등식 검증식 위치**: `veteran.ts:L344`의 `insuPrice = totalPrice - userPrice - mpvaPrice`는 CH12 §11.1의 검증식과 일치한다. 그러나 음수 보정 로직(L347-356) 이후 `pubPrice = insuPrice + mpvaPrice`로 재계산(L417)하므로, 통합 레이어에서 `pubPrice`의 정의를 `totalPrice - userPrice`로 혼용하면 불일치가 발생할 수 있다. ch08_analyst.md §6의 `pubPrice` 재정의 충돌 관찰과 같은 맥락이며, 통합 레이어에서 반드시 확인이 필요하다.

- **P06 13번·14번(공단 임직원, 병원장 재량)의 원외약국 적용 가능성**: CH12 §4.1은 이 두 항목이 "병원 내부 감면 운영 성격"이 강하다고 해석하나, 법령상 명시적으로 원외약국 제외를 규정하지 않는다. 정책 결정 시점까지 해당 케이스는 계산 엔진에서 런타임 경고 또는 미지원 오류를 반환하는 방어 로직을 두는 것이 안전하다.

- **2013.01.01 이전 공상등구분 7 처리**: CH12 §5.6은 2013.01.01부터 공상등구분 7→4로 전환한다고 명시한다. 즉 2013.01.01 이전 청구(레거시 재처리 등)에서는 코드 7이 유효하다. `veteran.ts`에는 해당 날짜 분기가 없으며, 현재는 실제 레거시 재처리 시나리오가 없을 것으로 추정되나 이력 데이터 처리를 지원할 경우 분기 추가가 필요하다.

- **CH12와 CH05~CH10 분리 원칙 준수 여부**: CH12 §12는 이 챕터가 "법령 PDF 기준 원천 규칙"이고 CH05~CH10이 "프로젝트 구현 해석"이라고 명확히 구분한다. 현재 `veteran.ts`는 두 역할을 한 파일에서 담당하고 있어, 향후 법령 변경 시 어느 부분을 수정해야 하는지 추적이 어려울 수 있다. 모듈 분리 설계를 검토할 것을 권장한다.
