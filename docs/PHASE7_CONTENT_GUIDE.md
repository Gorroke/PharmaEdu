## 1. 전환 원칙

### 핵심 철학: "왜 → 무엇 → 어떻게 → 직접 해보기"

CH 문서(기술 명세서)는 "다른 AI가 코드를 작성할 수 있도록" 쓰여졌습니다.  
레슨 콘텐츠는 정반대입니다. **"약국에 처방전 한 번 들고 가본 사람"이 읽고 이해하도록** 써야 합니다.

### 변환 규칙 5가지

#### 규칙 1 — 공식은 이야기로 먼저 온다

| CH 문서 방식 (금지) | 레슨 방식 (권장) |
|-----------------|----------------|
| `약품금액 = (int)(소모량 × 단가 + 0.5)` | "약값을 계산할 때는 먼저 '얼마나 쓰는지'와 '한 알 값'을 곱합니다. 그다음 원 단위 아래는 반올림하죠. 예를 들어 500원짜리 약을 하루에 3번, 7일간 먹는다면..." |
| 공식 먼저 → 설명은 주석 | 실생활 예시 먼저 → 공식은 "정리하자면" 박스 안에 |

**원칙**: 공식은 독자가 "아, 그거 이런 거구나!"라고 이미 느낀 후에야 보여줍니다.

#### 규칙 2 — 전문용어는 반드시 1~2문장으로 즉시 정의한다

| 금지 패턴 | 권장 패턴 |
|---------|---------|
| "4사5입 반올림을 적용합니다." | "4사5입 반올림(= 흔히 아는 반올림, 0.5 이상이면 올림)을 적용합니다." |
| "원미만 절사합니다." | "원 단위 아래 금액(예: 123.7원의 0.7원)은 버립니다. 이걸 '절사'라고 해요." |

**규칙**: 용어 등장 = 즉시 괄호 또는 다음 문장으로 정의. 용어집 별도 페이지 참조 유도는 금지.

#### 규칙 3 — 숫자는 항상 실제 숫자로

| 금지 패턴 | 권장 패턴 |
|---------|---------|
| "단가 × 투약량 × 횟수 × 일수" | "500원(단가) × 1정(1회 투약량) × 3번(하루 횟수) × 7일 = 10,500원" |
| "X라는 값이 Y보다 크면..." | "총약제비가 10,000원을 넘으면..." |

**규칙**: 추상적 변수명 금지. 항상 현실적인 숫자 예시로 대체합니다.

#### 규칙 4 — 소스코드 인용 금지 (예외: 심화 섹션)

CH 문서의 VB6, C# 코드, 의사코드(pseudocode)는 레슨 본문에 절대 인용하지 않습니다.  
꼭 필요하면 "더 깊이 알고 싶다면" 섹션에서 CH 문서 링크로만 유도합니다.

#### 규칙 5 — 소스 비교표는 학습 내용이 아니다

"비즈팜 vs EDB vs 유팜" 비교, "4소스 일치" 같은 기술 명세 요소는 레슨에 넣지 않습니다.  
학습자는 어떤 소프트웨어가 어떻게 구현했는지 알 필요가 없습니다.  
학습자가 알아야 할 것은 **건강보험 규칙 자체**입니다.

## 3. 글쓰기 스타일 가이드

### 3-1. 문체

- **기본**: 합쇼체와 해요체를 자연스럽게 혼용합니다.  
  - 설명 시: "~합니다", "~입니다"  
  - 독자에게 말 걸 때: "~해 보세요", "~생각해 보셨나요?"  
- **금지**: 논문체 (~임, ~함, ~되는바), 명령문 남용 (~하시오, ~할 것)
- **권장**: 학원 선생님이 칠판 앞에서 설명하는 투

예시:
```
❌ "약품금액은 단가에 소모량을 곱하여 원미만 4사5입으로 산출함."
✅ "약품금액은 단가에 소모량을 곱해서 구합니다. 원 단위 아래는 반올림하면 돼요."
```

### 3-2. 문단 길이

- 한 문단 = **3~5문장** 이내
- 2문장짜리 문단도 괜찮습니다
- 6문장 넘어가면 반드시 쪼개세요

### 3-3. 예시 규칙

- 각 핵심 개념마다 **최소 1개** 이상 예시 필수
- 예시에는 항상 실제 금액 사용 (500원, 10,000원, 3일 등)
- 가상의 환자 이름 사용 가능 ("60세 김영희 씨가 약국에 들어왔습니다...")
- 예시 ①과 예시 ②는 **다른 조건**이어야 함 (나이, 보험유형, 투약량 등)

### 3-4. 절대 금지 사항 (3대 금지)

**금지 1: 전문용어 갑작스러운 등장**
```
❌ "요양급여비용총액1에서 Insur_Amt를 빼고 trunc100을 적용합니다."
✅ "총 약제비에서 일부를 빼고, 100원 미만은 버립니다."
```

**금지 2: 수식만 나열하는 패턴**
```
❌ "공식: A = B × C × D. 단, E > 0일 때 F를 추가."
✅ 수식 전에 반드시 2~3문장 이야기로 맥락을 설명합니다.
```

**금지 3: CH 문서 복사-붙여넣기**
```
❌ (ch01-약품금액.md에서 표나 문단을 그대로 가져오는 것)
✅ CH 문서를 참고 자료로 읽되, 레슨은 완전히 새로 씁니다.
    학습자 관점에서 "처음 보는 사람에게 설명한다"고 생각하세요.
```

### 3-5. 격려와 공감 표현

어렵고 복잡한 내용일수록 독자를 배려하는 문장을 섞어야 합니다.

좋은 예:
- "처음엔 헷갈려도 괜찮아요. 예시를 같이 따라가면 금방 감이 옵니다."
- "여기까지 잘 따라오셨나요? 이게 오늘 가장 어려운 부분이에요."
- "복잡해 보이지만, 결국 세 단계로 요약됩니다."

## 4. 시각 요소 활용 기준

### 4-1. 비교표

언제: 두 가지 이상을 비교할 때, 조건별 결과가 다를 때  
형식: 마크다운 파이프 표

```markdown
| 구분 | 조건 | 본인 부담 |
|------|------|---------|
| 일반 성인 | 70세, 총약제비 15,000원 | 30% = 4,500원 |
| 65세 이상 (1구간) | 65세, 총약제비 8,000원 | 정액 1,000원 |
| 65세 이상 (2구간) | 68세, 총약제비 11,000원 | 20% = 2,200원 |
```

### 4-2. 단계 표시

순서가 있는 과정: 번호 리스트  
순서 없는 항목: 불릿 리스트

```markdown
약품금액 계산은 3단계로 진행됩니다:

1. **소모량 계산**: 1회 투약량 × 하루 횟수 × 투약 일수
2. **금액 계산**: 단가 × 소모량
3. **반올림**: 원 단위 아래는 반올림
```

### 4-3. 강조 요소

| 유형 | 마크다운 문법 | 사용 시점 |
|------|------------|---------|
| 인라인 강조 | `**중요 단어**` | 처음 등장하는 핵심 용어, 숫자 |
| 팁 | `> 💡 **팁**: 내용` | 실수 방지 요령, 외우면 편한 것 |
| 주의 | `> ⚠️ **주의**: 내용` | 자주 틀리는 포인트, 예외 케이스 |
| 정리 박스 | `> **정리**: 내용` | 공식, 규칙 최종 요약 |

사용 예:
```markdown
> 💡 **팁**: 65세 이상 여부는 조제일 기준으로 판단합니다. 생일이 지났는지 꼭 확인하세요.

> ⚠️ **주의**: 반올림과 절사는 다릅니다. 본인부담금은 '절사(버림)'를 씁니다.

> **정리**: 약품금액 = 단가 × 1회 투약량 × 하루 횟수 × 투약 일수 (원 단위 아래 반올림)
```

### 4-4. 계산 과정 표시

금액 계산 과정은 아래 형식으로 단계별로 보여줍니다:

```markdown
**계산 과정**

- 단가: 500원
- 1회 투약량: 1정
- 하루 횟수: 3번
- 투약 일수: 7일

→ 500원 × 1 × 3 × 7 = **10,500원**
```

## 6. 분량 가이드라인

| 기준 | 목표값 |
|------|-------|
| 레슨당 한국어 글자 수 | **2,000 ~ 4,000자** |
| 예상 읽기 시간 | **7 ~ 10분** |
| 섹션 수 | 5~8개 (템플릿 구조 준수) |
| 예시 개수 | 최소 2개 (예시 ①, 예시 ②) |
| 퀴즈 수 | 2~3개 |

### 분량 조정 기준

- **2,000자 미만**: 개념이 충분히 설명되지 않은 것. 예시를 추가하거나 "왜 필요한가" 섹션을 보강합니다.
- **4,000자 초과**: 두 개 레슨으로 분할합니다. 분할 기준: 독립적으로 이해 가능한 개념 단위.
- **퀴즈 1개**: 부족합니다. 반드시 2개 이상.

## 7. 선수 지식 수준 정의

레슨 작성 시 독자는 아래 수준이라고 가정합니다.

### 알고 있는 것 (전제 가능)
- 사칙연산 (더하기, 빼기, 곱하기, 나누기)
- 퍼센트 계산 (30% = 0.3, 1만원의 30% = 3,000원)
- 약국에 처방전을 들고 가서 약을 받아본 경험
- 건강보험이 있다는 것, 본인도 일부를 낸다는 것

### 모르는 것 (설명 필요)
- 요양급여비용, 급여/비급여 구분
- 정액/정률 본인부담의 차이
- 조제료가 별도로 청구된다는 것
- 4사5입 vs 절사(버림)의 차이
- 의료급여, 차상위, 산정특례 등 보험유형 구분

### 수학 수준
- 고등학교 수준 이하만 사용
- 루트, 지수, 통계 용어 사용 금지
- 복잡한 분수는 소수로 표현 (1/3 → "약 0.33")

```markdown
# Lesson 3: 약품금액, 이렇게 계산합니다

> 처방전에 적힌 약 하나하나의 금액이 어떻게 나오는지 배웁니다.

## 🎯 이 레슨의 목표
- 약품금액의 기본 계산 공식을 설명할 수 있다
- 단가, 투약량, 횟수, 일수의 관계를 이해한다
- 실제 숫자로 약품금액을 직접 계산할 수 있다

## 📚 시작하기 전에
이 레슨은 **Lesson 2: 보험 약가와 비급여 약가**를 읽고 오시면
"단가"가 어디서 오는지 이미 알고 계실 거예요.

## 1. 왜 필요한가?

약을 받을 때 영수증을 보면 약마다 금액이 적혀 있죠.
"타이레놀 7일치 — 3,500원" 이런 식으로요.
그 숫자, 어떻게 나오는 걸까요?

약국 직원이 손으로 계산하는 것도, 감으로 정하는 것도 아닙니다.
**건강보험 규정에 따라 정해진 공식**으로 계산합니다.
이 공식을 알면, 영수증을 보고 "맞게 계산됐네"라고 확인할 수 있어요.

## 2. 핵심 개념

약품금액을 결정하는 요소는 딱 4가지입니다.

- **단가**: 그 약 1개(또는 1mL, 1회 용량)의 보험 가격
- **1회 투약량**: 한 번에 얼마나 복용하는지 (예: 1정, 2정, 0.5정)
- **1일 투여 횟수**: 하루에 몇 번 먹는지 (예: 하루 3번)
- **총 투약 일수**: 며칠치 처방인지 (예: 7일분)

### 예시 ①: 기본 계산

> 500원짜리 약을 하루 3번, 7일간 처방받았다면?

- 단가: 500원
- 1회 투약량: 1정
- 1일 투여 횟수: 3번
- 총 투약 일수: 7일

→ **500원 × 1 × 3 × 7 = 10,500원**

### 예시 ②: 절반 용량 처방

> 800원짜리 약을 하루에 반 정(0.5정)씩, 하루 2번, 5일간 처방받았다면?

→ **800원 × 0.5 × 2 × 5 = 4,000원**

## 3. 어떻게 작동하는가?

공식을 정리하면 이렇습니다.

> **정리**: 약품금액 = 단가 × 1회 투약량 × 1일 투여 횟수 × 총 투약 일수

한 가지만 더 알아두면 됩니다. 계산 결과에 원 단위 아래 금액이 생기면
**반올림**(0.5원 이상이면 올림, 미만이면 버림)합니다.

> 💡 **팁**: 0.5원처럼 아주 작은 금액이 생기는 건 투약량이 소수점일 때입니다.
> (예: 0.3정 × 5 = 1.5정처럼 나오는 경우)

> ⚠️ **주의**: 조제료는 여기 포함되지 않습니다. 약품금액과 조제료는 별도로 계산해요.
```
