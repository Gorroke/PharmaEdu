### Lesson 1 — 약제비란 무엇인가?

| 항목 | 내용 |
|------|------|
| **번호** | 1 |
| **제목** | 약제비란 무엇인가? |
| **부제** | 약국에서 청구하는 돈, 어떻게 계산될까? |
| **대상** | 약국 업무 입문자, 보험 청구에 처음 관심 생긴 일반인 |
| **선수 레슨** | 없음 |
| **예상 시간** | 15분 |
| **파일 경로** | `src/content/lessons/lesson-01-what-is-yakjaebi.md` |

**학습 목표 (3개)**
1. 약제비가 '약값'과 '조제료'로 구성된다는 것을 설명할 수 있다.
2. 건강보험에서 약제비가 환자·공단·국가로 3자 분담된다는 구조를 이해한다.
3. 약국이 공단에 청구하는 금액과 환자가 내는 금액의 차이를 구분할 수 있다.

**핵심 개념 (6개)**
- 약제비 (약가 + 조제료)
- 요양급여비용 (공단이 인정하는 급여 총액)
- 본인부담금 (환자 부담분)
- 공단부담금 (국민건강보험공단 지급분)
- 심사청구 (HIRA/공단에 청구하는 행위)
- 급여·비급여 구분

**원본 참조 챕터**
- CH00 (§1 계산 엔진 입력 개요) — 약제비 구성 전체 그림
- CH05 (§1 개요) — 본인부담 흐름 요약

**연결 퀴즈 카테고리** | `quiz_yakjaebi_basics`

**계산기 프리셋** | `preset_simple_1drug` — 약품 1개, 건강보험, 야간·가산 없음

## 3. Phase 7B 공통 스타일 가이드 (레슨 작가 참조)

> **핵심 원칙: 이 레슨은 스펙 문서가 아닌 교육 콘텐츠다.**  
> CH 문서를 복붙하는 것은 절대 금지. 내용을 완전히 소화하고 학습자 언어로 재서술한다.

### 3-1. 어투와 톤

| 금지 | 권장 |
|------|------|
| "산정 방식은 아래와 같다." | "이렇게 생각해보세요." |
| "§1.2에 따르면 ..." | "왜 이 정보가 필요할까요?" |
| 수동형 나열 | 질문 → 답 구조 |
| 코드 블록 남발 | 먼저 개념, 이후 코드 예시 |

**권장 어투 예시:**
- "약품금액 계산은 사실 굉장히 단순합니다. 단가 × 수량 × 횟수 × 일수. 그런데 왜 청구 오류가 생길까요? 바로 '반올림 위치' 때문입니다."
- "처방전을 보면 뭔가 많이 적혀 있죠? 실제로 계산에 쓰이는 건 이 중 7가지입니다."

### 3-2. 레슨 구성 순서 (템플릿)

```
## [레슨 제목]

### WHY — 왜 알아야 하나요?
(학습자가 왜 이걸 배워야 하는지 1~2문단, 실제 상황 연결)

### 핵심 개념 훑어보기
(3~5개 개념 카드 형식 — 개념명 + 한 줄 설명)

### 이해하기 (본문)
(사례 → 설명 → 공식/규칙 순서로. 표·다이어그램 적극 활용)

> [CHECKPOINT] 여기까지 이해했나요? ← 중간 이해도 체크 포인트

### 예제로 확인하기
(실제 처방전 사례 1개로 직접 계산)

### 정리
(핵심 3줄 요약 + 다음 레슨 예고)
```

### 3-3. 표·다이어그램 사용 규칙

- 비교가 필요한 경우: **반드시 표** 사용 (텍스트 나열 금지)
- 흐름이 있는 경우: **ASCII 화살표 다이어그램** 사용
- 공식이 있는 경우: 코드 블록(`math` 또는 `python` 표기) 사용
- 표의 열은 최대 5개 이하로 제한

### 3-4. CHECKPOINT 규칙

- 레슨당 **2~3개** CHECKPOINT 배치 (레슨 중간부, 말미)
- 형식: `> [CHECKPOINT] 질문 내용`
- 예: `> [CHECKPOINT] Z코드 Z4107의 '4107'은 무엇을 뜻하나요?`
- 답은 CHECKPOINT 바로 아래 `<details>` 토글 또는 별도 섹션으로

### 3-5. 금지 사항

- CH 문서의 표·수식 직접 복붙 금지
- 소스 레퍼런스 약어 (`[비즈팜]`, `[EDB]` 등) 본문 노출 금지
- 44개 파라미터 전체 나열 금지 (학습 목표에 필요한 것만 선택)
- 구현 레벨 코드(C#/TypeScript 실코드) 본문 삽입 금지 (개념 설명에 집중)

## 4. Phase 7C 인터랙티브 컴포넌트 데이터 구조

Phase 7C에서 구현할 인터랙티브 컴포넌트가 참조할 TypeScript 타입 정의.

```typescript
// src/types/lessons.ts

export type LessonLevel = '기초' | '중급' | '심화';

export type LessonMeta = {
  number: number;                    // 1~10
  slug: string;                      // 'lesson-01-what-is-yakjaebi'
  title: string;                     // '약제비란 무엇인가?'
  subtitle: string;                  // 한 줄 부제
  level: LessonLevel;
  prerequisites: number[];           // 선수 레슨 번호 배열
  estimatedMinutes: number;          // 15~30
  objectives: string[];              // 학습 목표 3~5개
  keyConcepts: string[];             // 핵심 개념 5~10개
  sourceChapters: string[];          // ['CH00', 'CH05'] 등
  quizCategory: string;              // 'quiz_yakjaebi_basics'
  calculatorPreset: string;          // 'preset_simple_1drug'
};

export type LessonProgress = {
  lessonNumber: number;
  isRead: boolean;           // 레슨 본문 스크롤 완료
  isQuizPassed: boolean;     // 지식 체크 통과 (70% 이상)
  completedAt: string | null; // ISO 8601
};

export type LearningState = {
  progress: LessonProgress[];   // 10개 레슨 진도
  currentLesson: number | null;
  totalCompletedCount: number;
  lastActivityAt: string | null;
};

// localStorage 키: 'pharmaEdu_learningState'
```

### 레슨 언락 규칙

```
레슨 N이 언락되는 조건:
  - Lesson 1: 항상 언락
  - Lesson N (N ≥ 2): prerequisites 배열의 모든 레슨이 isQuizPassed === true

완료 조건 (한 레슨):
  1. isRead = true (스크롤 90% 이상 도달)
  2. isQuizPassed = true (퀴즈 70% 이상 정답)
  → completedAt 기록, 다음 레슨 언락
```

### 계산기 프리셋 데이터 구조

export type CalculatorPreset = {
